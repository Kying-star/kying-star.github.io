#  React Fiber



## Fiber 解决了什么问题？

### CPU的瓶颈（Time Slicing 时间分片）

当项目变得庞大、组件数量繁多、遇到**大计算量**的操作或者**设备性能不足**使得页面掉帧，导致卡顿。

### IO的瓶颈（Suspense）

发送网络请求后，由于需要等待数据返回才能进一步操作导致不能快速响应。如果响应时间过长，那么白屏的时间就会很影响用户体验。



## 为什么要使用 React Fiber？

在 vue2 和 reactV15 中都存在当组件树过大和网络延迟过大时，卡顿和白屏的问题。
